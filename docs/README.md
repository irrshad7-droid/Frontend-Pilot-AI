# FrontendPilot AI — Documentation Index

This directory serves as the definitive source of truth for the project's architecture, design decisions, and engineering handoffs. It is specifically structured for long-term development and LLM agent contexts (such as ChatGPT Codex).

## Directory Structure

### `/architecture`
Contains core design specifications and implementation plans.
*   **`implementation_plan.md`**: The complete architectural specification covering the Explorer, Source Mapper, Analyzer, Repair, and Verifier agents. Details all schemas, validation rules, and strict design constraints for the backend intelligence.

### `/handoffs`
Contains repository state snapshots and historical progress tracking.
*   **`repository_inventory.md`**: The synchronized inventory of backend modules, FastAPI routes, schemas, dependencies, and the working-tree state captured during the latest documentation pass.
*   **`walkthrough.md`**: A chronological record of accomplishments, summarizing how the pipeline was incrementally built and tested across each project phase.
*   **`task.md`**: The granular checklist tracking every phase of the project and identifying outstanding tasks for future execution.

### `/demo`
Reserved for visual assets, recordings, and examples demonstrating the autonomous workflow in action.
